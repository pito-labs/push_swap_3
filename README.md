# push_swap_3
アルゴリズム:LIS データ構造: 双方向連結リスト

## データ構造: 
双方向循環連結リスト (Doubly Circular Linked List)
t_node 構造体を使用し、bool is_lis というフラグを追加。
これで「LISの一部であるノード」を識別。

## アルゴリズム: 
LIS (最長増加部分列) + Greedy法
LISの計算: 最初にスタックAの中で「増加し続けている最も長い部分列（LIS）」を計算し、見つけ出します。（O(N²)のDPなどで実装）
### 選択的プッシュ: 
push_swap_3 が無条件にすべてBへ送っていたのに対し、このバージョンは 「LISに含まれないノードだけ」 をBへプッシュします。
これにより、すでに整列されている長い列をAに温存できるため、戻す手間が大幅に減ります。
Greedyな戻し (Turk Algo): Bにある要素をAに戻す際は、push_swap_3 と同様に計算コスト（命令数）が最小になるペアを探して戻します
